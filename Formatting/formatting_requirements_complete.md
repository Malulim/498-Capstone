# Complete Formatting Requirements (Consolidated)

This document consolidates the OCR-extracted requirements from the provided `formatting.md` source images into a single readable guide for general readers.

## 1) Document Scope

The source appears to include requirements for at least two related deliverables:

1. **Project Specifications and Risk Assessment document**
2. **Detailed Design and Project Timeline document**

Where rules differ by deliverable (for example, page limits), this guide calls that out explicitly.

## 2) Required Title Page Content

Include the following on the title page:

- Group number (example format shown in source: `2026.75`)
- Student details: names, student IDs, and email addresses
- Name of consultant
- University / faculty / department naming where requested:
  - University of Waterloo
  - Faculty of Engineering
  - Department of Electrical and Computer Engineering
- Project title
- Submission date

## 3) Table of Contents and Page Numbering

- Include a **Table of Contents** on a separate page.
- Number all pages **after** the title page.

## 4) Project Specification Requirements

For the Project Specifications section:

- Present specifications in **table form** (with added text as needed).
- Label each specification as:
  - **Essential**, or
  - **Non-essential**
- Each specification must be:
  - **Verifiable**
  - **Appropriate**
- The complete set of specifications must be:
  - **Complete**
  - **Open-ended** (can be implemented in multiple valid ways)

## 5) High-Level Description and Block Diagram

For the high-level project description:

- Include a **block diagram** that gives a big-picture system view.
- The diagram should be clearly labeled and show:
  - Inputs
  - Subsystems
  - Outputs
- Target around **4-6 subsystems**.

## 6) Writing Style and Technical Communication

Use formal technical writing throughout:

- Formal language and professional layout/font
- Prefer **present tense** in most places
- Define acronyms/jargon on first use
- Include units for numeric values (for example, `m`, `V`)
- Correct spelling and grammar
- Clear structure, flow, and paragraph quality
- Minimize jargon when audience expertise may vary

## 7) Figures, Tables, and References

### Figures

- Number each figure.
- Put figure caption **below** the figure.
- Cite each figure in body text.
- Keep figure numbering consistent.
- Place figures close to first mention.
- Keep figures readable; avoid splitting across pages when possible.

### Tables

- Number each table.
- Put table caption **above** the table.
- Cite each table in body text.
- Keep table numbering consistent.
- Place tables close to first mention.

### References

- Use **IEEE** reference formatting.
- Use numbered citations like `[1]`, `[2]`, etc.
- If a figure/table is taken from another source, cite that source in the caption.
- Every entry in References should be cited at least once in the report body.

## 8) Font, Spacing, and Margins

General formatting requirements extracted from source:

- Main text font: **11 or 12 pt**
- Line spacing: **1.15 to 1.50**
- Margins: at least **1 inch (2.54 cm)** on all sides
- Text inside figures: at least **9 pt**
- Any standard professional-looking font is acceptable

Exception noted in source:

- **Single-line spacing** may be used for:
  - Table of Contents
  - References

## 9) Page Limit Rules (By Deliverable)

The OCR source includes two page-limit statements for different deliverables:

1. **Project Specifications and Risk Assessment**
   - Page limit shown: **10 pages**, including all content (title page, table of contents, references, etc.).

2. **Detailed Design and Project Timeline**
   - Explicit exception: page limit is **30 pages**, including everything (title page, table of contents, references, optional appendices).

Use the page limit that matches the specific deliverable being submitted.

## 10) Practical Submission Checklist

Before submission, verify:

- Title page includes all required identities and metadata
- Table of contents is present and page numbering is correct
- Specs are tabulated, labeled (essential/non-essential), and verifiable
- Block diagram is complete and well labeled
- Figures/tables are numbered, captioned, cited, and readable
- References are IEEE-style and every reference is cited in text
- Font/spacing/margins satisfy required ranges
- Correct page limit is applied for the target deliverable

## 11) Note on Source Quality

This consolidation is based on OCR of image-based pages. Minor OCR artifacts may exist in wording, but the requirement intent has been normalized for readability and consistency.
